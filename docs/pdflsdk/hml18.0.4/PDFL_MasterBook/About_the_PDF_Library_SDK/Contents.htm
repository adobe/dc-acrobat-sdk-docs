The latest PDF Library SDK consists of the following components:

•Core libraries that provide the functionality

•Header files that provide access to the libraries

•Fonts used in the library’s basic operations

•Sample applications and code snippets showing how to use the library for a variety of purposes

•Documentation of installation and development techniques and of the PDF Library APIs

The components that are shipped with the PDF Library SDK are listed in Developing Plug-ins and Applications. These include the PDF Library, helper libraries, headers, and samples.

SnippetRunner

The SnippetRunner is based on the SnippetRunner provided with the Acrobat 8 SDK, but is adapted to the PDF Library environment. It has a command-line interface and prints output to the console. It is found in the SnippetRunner subdirectory of the Samples directory. For more information, see the Snippet Runner Cookbook.

The following table shows the commands that you can use to control the snippet runner in a command shell. The snippets (see PDF Snippets) are arranged in a directory-like hierarchy. You can use the findsnip command to locate a snippet and the cd command to switch between directories.

Note:The command names are not case-sensitive. However, the snippet names and directory names are case-sensitive.

Snippet Runner commands

Command

Description

cd

Navigate the hierarchy of snippets using standard file path semantics.

findsnip

Search for a snippet based on a specified substring of the snippet identification string. If the result is a single file, prompts for execution.

snipinfo

Display the description of a specified snippet.

docname

Display the name and file information for the current document.

setpage

Specify the current page of the document.

opendoc

Open the specified document. This document becomes the new current document. If the previously open document was modified, the command prompts to save changes.

closedoc

Close the current document. If it was modified, the command prompts the user to save changes.

savedoc

Save the document to the specified file.

revertdoc

Revert the current document to its last saved condition.

run

Run the specified snippet. The snippet must exist in the current directory—you cannot specify a path.

ls

List the snippets (and hierarchy nodes) in the current directory.

pwd

Print the current directory.

help

List all the commands with their synopses, or provide help for a specified command.

quit

Quit the application.

whichPage

Display the current selected page.
