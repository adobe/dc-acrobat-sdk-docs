Raising exceptions

Although many Acrobat core API methods raise exceptions, some methods do return error codes or NULL if something goes wrong. Inside a DURING HANDLER block, if your plug-in detects one of these cases, it can invoke the ASRaise method, which generates an exception. This has the effect of causing the HANDLER END_HANDLER code to execute, just as if the original method raised an exception.

You can also use the RERAISE macro when you do not want your exception handler to handle an exception, but want to pass the exception to the next exception handler on the stack.

Note:Your plug-in should use the ASRegisterErrorString method to define its own exceptions.
