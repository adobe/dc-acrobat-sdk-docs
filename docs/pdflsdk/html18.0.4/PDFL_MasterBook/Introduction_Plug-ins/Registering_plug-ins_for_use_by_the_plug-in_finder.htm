Registering plug-ins for use by the plug-in finder

You can register your plug-in with Adobe to ensure that users are prompted to download and install it when they open PDF documents that require it.

Registering your plug-in means that Adobe adds information about your plug-in to a list of registered plug-ins. Each entry in this list associates a PDF dictionary extension and other characteristics with a URL from which users can download and install the plug-in that processes that extension. Acrobat and Adobe Reader access the list over the web. The list is not publically displayed on adobe.com. That is, it cannot be used to advertise the availability of your plug-ins. If you wish to post on adobe.com information about your plug-in , see your Adobe reprsentative.

How Acrobat uses the plug-in list

When Acrobat open a PDF document that contains a PDF extension that it doesn’t recognize, it goes through the following steps:

1.Examines the plug-ins that have registered with it to determine whether one of them can process the dictionary. If Acrobat finds one, it invokes that plug-in and the next step is skipped.

2.Examines the list of registered plug-ins to determine whether one of them can process the dictionary, and responds as follows:

•If Acrobat finds a registered plug-in that can process the PDF extension, it directs the user to the URL associated with the URL. Ideally, the website referenced by the URL provides download instructions for the plugin.

•If Acrobat cannot find a registered plug-in that can process the PDF extension, it directs the user to the generic website pictured below.

 

Submitting a request to register your plug-in

You can add your plug-in to the file that plug-in finder checks by sending an email to AcrobatTechnicalEvangelist@adobe.com. Include in your email the characteristics of your plug-in and the fully qualified URL where you want the end-user to be redirected. The following is an example of the information supplied

prod=Reader

os=Win

type=DigSig

name=MYCO_aSignatureHandler

lang=SVE

url=http://www.myPluginDownloadSite.htm

The following table describes the values expected for each property.

Property name

Description and values

prod

(Optional) Product supported by the plug-in. Values include:

•   Reader

•   Acrobat

If this property is omitted, plug-in finder assumes the plug-in supports both Adobe Reader and Acrobat

os

(Optional) Operating system. Values include:

•Win

•Mac

•Unix

If this property is omitted, plug-in finder assumes the plug-in supports all operating systems.

Note:Only Adobe Reader supports the Unix platform.

type and name

Type of service provided by the plug-in. Values include:

•Filter: The name key corresponds to the value of the Filter key in a stream dictionary.

•DigSig: The name key corresponds to the value of the Filter key in a signature field seed value dictionary.

•Annot: The name key corresponds to the value of the Subtype key in an annotation dictionary. For example, name can be Widget, Circle, and Stamp.

Other types are supported. For more information, see your Adobe representative.

lang

(Optional) Language supported by the plug-in. Values use the 3-character convention. For example, KOR (Korean), SVE (Swedish), and DEU (German). If lang is omitted, plug-in finder assumes the plug-in supports all languages.

url

URL for a website that provides download instructions for the plugin
