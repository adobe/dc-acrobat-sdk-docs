SRV DNS record

The SRV record allows the Service Name to be bound to a specific IP address and port corresponding to the service. In BIND the entry would appear as follows:

   <Service Name> IN SRV 0 0 <port #> <Service Target>

If the service above were running on port 8080 of the machine exampleService, the entry would appear as follows:

   Example\032Service\032Name IN SRV 0 0 8080 exampleService.corp.example.com

TXT DNS record

The TXT record allows the Service Name to be bound to a set of named parameters that are specific to the service you are publishing, and are described in the following table.

Service type

TXT parameters

_acrobatSRV._tcp

txtvers: The version number of the service. This is 1 for Acrobat 8.0.

path: The path portion of the URL for the service. For example, if the URL is:

http://example.org/services/ExampleService

then the path parameter would be set to:

services/ExampleService.

feedType: The Service Type (either "RSS" or "JS"). "RSS" indicates that it is an RSS feed to be deployed in Tracker, and "JS" indicates that it is an Initiation Wizard.

_acrobatRMACR._tcp

txtvers: The version number of the service. This is 1 for Acrobat 8.0.

setting: The collaboration store setting. This depends on the type of repository being used.

type: The collaboration service type. The possible values are "DAVFDF‚Äù (WebDAV), "FSFDF" (Network Folder), and "CONFIG" (Custom Server Type).

In BIND the entry would appear as follows:

   <Service Name> IN TXT "<p1 name>=<p1 value>" "<p2 name>=<p2 value>" ...
   "<pN name> = <pN value>"

For example, if the service above is an Acrobat service with a feedType of RSS and a path of RSS/example.rss, then the entry would appear as follows:

   Example\032Service\032Name IN TXT "txtvers=1" "feedType=RSS"
   "path=RSS/example.rss"
