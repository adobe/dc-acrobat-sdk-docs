Adding message handling

Plug-ins can add their own DDE messages and Apple events to those supported by Acrobat and Acrobat Reader. On Windows, plug-ins can register to receive DDE messages directly. On Mac OS, plug-ins must hook into Acrobat or Acrobat Reader’s Apple event handling loop to handle Apple events. To do this, replace the API’s AVAppHandleAppleEvent method. For information about replacing methods, see Replacing HFT methods.

If a plug-in receives an Apple event it does not want to handle, it should invoke the implementation of the method it replaced, allowing other plug-ins or Acrobat or Acrobat Reader the opportunity to handle the Apple event. Acrobat and Acrobat Reader on UNIX do not currently provide built-in IAC support, but plug-ins can add IAC support via RPC or other mechanisms.

Note:Plug-ins should use the DDEML library to handle DDE messages. Problems may arise if they do not.
