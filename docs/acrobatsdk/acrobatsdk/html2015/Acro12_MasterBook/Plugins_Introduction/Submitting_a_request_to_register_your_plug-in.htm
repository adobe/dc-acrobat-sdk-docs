Submitting a request to register your plug-in

You can add your plug-in to the file that plug-in finder checks by sending an email to AcrobatTechnicalEvangelist@adobe.com. Include in your email the characteristics of your plug-in and the fully qualified URL where you want the end-user to be redirected. The following is an example of the information supplied

prod=Reader

os=Win

type=DigSig

name=MYCO_aSignatureHandler

lang=SVE

url=http://www.myPluginDownloadSite.htm

The following table describes the values expected for each property.

Property name

Description and values

prod

(Optional) Product supported by the plug-in. Values include:

  •    Reader

  •    Acrobat

If this property is omitted, plug-in finder assumes the plug-in supports both Acrobat Reader and Acrobat

os

(Optional) Operating system. Values include:

  • Win

  • Mac

  • Unix

If this property is omitted, plug-in finder assumes the plug-in supports all operating systems.

Note:Only Acrobat Reader supports the Unix platform.

type and name

Type of service provided by the plug-in. Values include:

  • Filter: The name key corresponds to the value of the Filter key in a stream dictionary.

  • DigSig: The name key corresponds to the value of the Filter key in a signature field seed value dictionary.

  • Annot: The name key corresponds to the value of the Subtype key in an annotation dictionary. For example, name can be Widget, Circle, and Stamp.: The name key corresponds to the value of the Filter key in an encryption dictionary.

  • Plugin: The name key corresponds to the value of a  dictionary extension. (Plugin&name=PDFXML)

  • Update: The name key corresonds to the value of ??????????. (Update&name=EBX_HANDLER)

  • Widget. The name key corresponds to the value of ???. (Widget&name=Signature)

  • ? The name key corresponds to the value of the FS key in a file specification dictionary.

Other types are supported. For more information, see your Adobe representative.

lang

(Optional) Language supported by the plug-in. Values use the 3-character convention. For example, KOR (Korean), SVE (Swedish), and DEU (German). If lang is omitted, plug-in finder assumes the plug-in supports all languages.

url

URL for a website that provides download instructions for the plugin

Acrobat tries to find the plug-in with the most matching properties. If multiple plug-ins have some of the same type and name,

The following characters must not be used in the name field: colon (&), equal sign (=), and ampersand (&).
