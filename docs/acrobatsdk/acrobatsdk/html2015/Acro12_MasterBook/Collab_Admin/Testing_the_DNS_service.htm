Testing the DNS service

The service should now be advertised using DNS. To test that it has been advertised properly, you can use nslookup.

To find the services for a specific service type, use the following syntax:

   nslookup -q=ptr <Service Type>.<Default DNS Domain>

For example, to search for an Acrobat service in the domain corp.example.com, use the following syntax:

   nslookup -q=ptr _acrobatSRV._tcp.corp.example.com

This will return the set of service names that are advertised as Acrobat services. To obtain the SRV and TXT records for a specific service name, use the following syntax:

   nslookup -q=any <Service Name>.<Default DNS Domain>

For example, to obtain the SRV and TXT records for a service named “My Example Service”, use the following syntax:

   nslookup -q=any My\032\Example\032Service.corp.example.com
